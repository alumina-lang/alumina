  • fn compiler_fence(ordering: Ordering)

    source

    Compiler-only memory barrier

    This prevents the compiler from reordering memory accesses according to the memory access ordering, but does not prevent hardware reordering.

  • fn spin_loop()

    source

    A hint to the CPU that we are spinning

    This does not result in a system call like yield, but can reduce power usage or allow other hyperthreads to run.

    Example

    use std::sync::{Atomic, Ordering, spin_loop};

let flag = Atomic::new(false);
while !flag.exchange(true, Ordering::Acquire) {
    spin_loop();
}
